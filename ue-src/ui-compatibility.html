placeloder

placeholder是HTML5中的新属性，提供可描述输入字段预期值的提示信息（hint），该提示会在输入字段为空时显示，并会在字段获得焦点时消失，为页面的设计带来极大的方便。 但是，众所周知，IE6~IE9都不支持HTML5 placeholder，我们要想渐进使用placeholder, 对于IE浏览器，必须要借助JavaScript交互了。

在组件中我们使用了jQuery placeholder插件对ie8、ie9的占位符进行处理，下面是我们处理的初始源代码：

	            	
var JPlaceHolder = {
	//检测input中是否支持placeholder属性
	_check : function(){
        return 'placeholder' in document.createElement('input');
    },
    //初始化
    init : function(){
        if(!this._check()){
            this.fix();
        }
    },
    //修复
    fix : function(){
    	//遍历
        $(':input[placeholder]').each(function(index, element) {
            var self = $(this), 
            	txt = self.attr('placeholder');
            //在input标签外面包一层div
            self.wrap($('
').css({position:'relative', zoom:'1', border:'none', background:'none', padding:'none', margin:'none'}));
            var pos = self.position(), 
            	h = self.outerHeight(true), 
            	paddingleft = self.css('padding-left');
            //创建span元素，.text()：设置占位符内容，zIndex：设置层级
            var holder = $('').text(txt).css({position:'absolute', zIndex:'3', left:pos.left, top:pos.top, height:h, lineHeight:h+'px', paddingLeft:paddingleft, color:'#ccc'}).appendTo(self.parent());
            //获得焦点，隐藏占位符
            self.focusin(function(e) {
                holder.hide();
            //失焦判断为空后显示占位符
            }).focusout(function(e) {
                if(!self.val()){
                    holder.show();
                }
            }).change(function(e){
            	if(!self.val()){
            		holder.show();
            	}else{
            		holder.hide();
            	}
            });
            //span元素被单击时，隐藏span元素，input获取焦点
            holder.click(function(e) {
                holder.hide();
                self.focus();
            });
        });
    }
};
//初始化
$(function(){
    JPlaceHolder.init();
});

圆角问题在ie8中也是一个很重要的问题。我们都知道IE浏览器暂不支持CSS3的一些属性，除了圆角问题还有盒阴影(box-shadow)，背景渐变(gradient)，多图片背景(multiple background images) 等的问题，都是ie8浏览器下的疑难杂症。这里主要说一下圆角问题。

解决ie8圆角问题的方法有多种，但是我们这里主要说的是用pie的方式解决圆角问题。
pie的解决方式有两种，一个是pie.htc（不建议使用），另外一个是pie.js。

1. pie.htc

使用方法：
（1）首先在和html同文件夹下加载pie.htc文件。
（2）css 中使用 behavior 来加载即可自动实现效果:

                   
#test {
    border-radius: 3px;
    behavior: url(PIE.htc);
}

注意：
（1）IE浏览器的behavior 属性是相对于HTML文档而言的，与CSS其他的属性不一样，不是相对于CSS文档而言的。这使得使用pie.htc文件不怎么方便。 如果绝对路径于根目录，则CSS文件不方便移动；如果相对路径与HTML文档，则pie.htc文件在不同HTML页面见的重用性大大降低。同时，诸如border-image后面的URL属性路径也不好处理。
（2）提供正确的Content-Type
要想让IE浏览器支持htc文件，需要一个有着"text/x-component" 字样的content-type 头部，否则，会忽视behavior。绝大数web服务器提供了正确的content-type，但是还有一部分则有问题。例如的我的空间域名商就没有"text/x-component"字样的content-type，可能是出于安全的考虑。

我们并不提倡使用这种方法，因为pie.htc与页面的耦合性太高。在组件中我们使用另外一种很有效的方法：pie.js。

需要注意的地方：

（1）z-index相关问题

    IE下这些CSS3效果实现是借助于VML，由VML绘制圆角或是投影效果的容器元素，然后这个容器元素作为目标元素的后兄弟节点插入，如果目标元素position:absolute 或是 position:relative，则这个css3-container元素将会设置与之一样的z-index值，在DOM tree中，同级的元素总是后面的覆盖前面的，所以这样就实现了覆盖，又避免了可能有其他元素正好插入其中。 所以，问题来了，如果目前元素的position属性为static，也就是默认属性，则z-index属性是没有用的，无覆盖可言，所以此时IE浏览器下CSS3的渲染是不会成功的。要解决也很简单，设置目标元素position:relative或是设置祖先元素position:relative并赋予一个z-index值（不可为-1）。

（2）缩写的问题

    使用PIE实现IE下的CSS3渲染（其他方法也是一样），只能使用缩写的形式，例如圆角效果，我们可以设置border-top-left-radius表示左上圆角，但是PIE确实不支持这种写法的，只能是老老实实的缩写。

如果你想使用background-size: cover设置全屏背景，很遗憾IE8办不到...但可以使用IE独有的AlphaImageLoader滤镜来实现，添加一条如下的CSS样式：

将sizingMethod设置为scale就OK了。

CSS3中提供支持滤镜效果的属性filter，比如支持高斯模糊效果的blur（类似iOS7的效果）：

rgba及透明度

我们平时使用最多的设置透明度方法是：rgba和opacity。

而RGBA颜色归为CSS3属性，其余RGB的区别在于多了个A，也就是Alpha透明度，使用rgba无论是背景，边框，文字颜色等都可以用半透明实现了，然而rgba与opacity都属于CSS3属性，ie8并不支持。

我们以下面代码为例：

                    
background: rgba(200, 54, 54, 0.5);
或
background: rgb(200, 54, 54);
opacity: 0.5;

这时候我们可能会想起IE的半透明滤镜：

                    
filter: alpha(opacity=50);

但是加上这个半透明滤镜的话你会发现没有什么效果，因为filter:alpha(opacity=50);并不是设置背景透明度的，但它可以设置边框，字体等的透明度。其中opacity的值取0~100之间。

如何使背景透明？

我们在组件中使用的是IE渐变滤镜，最简单是使用如下：

                    
filter: progid:DXImageTransform.Microsoft.Gradient(startColorStr = #AARRGGBB,endColorStr = #AARRGGBB)

startColorStr:可选项。字符串(String)。设置或检索色彩渐变的开始颜色和透明度。

其格式为 #AARRGGBB 。 AA 、 RR 、 GG 、 BB 为十六进制正整数。取值范围为 00 – FF 。 RR 指定红色值， GG 指定绿色值， BB 指定蓝色值， AA 指定透明度。 00 是完全透明。 FF 是完全不透明。超出取值范围的值将被恢复为默认值。

取值范围为 #FF000000 – #FFFFFFFF 。默认值为 #FF0000FF 。不透明蓝色。

EndColorStr:可选项。字符串(String)。设置或检索色彩渐变的结束颜色和透明度。默认值为 #FF000000 。不透明黑色。

所以rgba+filter设置背景透明度如下：

                   
background: rgba(0,0,0,0.5);
filter: progid:DXImageTransform.Microsoft.gradient(startcolorstr = #7F000000,endcolorstr = #7F000000);

transform

中所周知，transform属于css3中的属性，所以说此属性在ie8中并没有什么作用。transform的功能是使图片之类的元素具有旋转功能，比如：

                   
-webkit-transform: rotate(-90deg);
-moz-transform: rotate(-90deg);
-o-transform: rotate(-90deg);
-ms-transform: rotate(-90deg);
transform: rotate(-90deg);

这段代码的功能是将所选元素逆时针旋转90度。但是怎么才能让ie8也支持这种功能呢，这时候就必须想到filter了。

                   
filter:progid:DXImageTransform.Microsoft.BasicImage(rotation=3);

这句话是处理ie8不支持css3中的transform属性。其中rotation=3是关键，这里的参数可以是0,1,2,3旋转的角度只要将这些数值乘以90(π/2)就可以了，所以呢"rotation=3"表示顺时针旋转270度，与transform:rotate(270deg); 是一个意思。所以，这里，就会有些小小的局限——不能实现任意角度的旋转，只能是90度，180度以及270度。但是，IE浏览器不是个简单的罗罗，要实现任意角度的旋转，它还是有办法的，只是要比上面的麻烦些，难理解些，要用到矩阵变换滤镜

IE8浏览器（在非标准模式下），在CSS中，要使用“-ms-filter”代替“filter”。要注意只是在非标准模式下。

以下代码将实现图片顺指针旋转60度的效果：

           
filter:progid:DXImageTransform.Microsoft.Matrix(M11=0.5,M12=-0.866,M21=0.866,M22=0.5,SizingMethod='auto expand');

M11=0.5,M12=-0.866,M21=0.866,M22=0.5计算方法：

假设我们要旋转的角度是rotation，则计算过程如下：

                	
sin = sin(roation);cos = cos(roation);
M11 = cos;M12 = -sin;M21 = sin;M22 = cos;

例如这里要旋转60度，即rotation=60，所以sin = sin(60) = 0.866, cos = cos(60) = 0.5，于是就有了"M11=0.5,M12=-0.866,M21=0.866,M22=0.5"。 所以呢，要实现IE下任意角度图片的旋转记住下面的式子就可以了：filter:progid:DXImageTransform.Microsoft.Matrix(M11=cos(roation),M12=-sin(roation),M21=sin(roation),M22=cos(roation),SizingMethod='auto expand');

但是需要注意的是：
如果filter想写在:after中的话，可能会不产生任何效果，所以应该尽量避免将filter写在:after，:before等伪元素下。



::before ::after

::after是一个CSS伪元素，使用::after，你可以从CSS里往页面上新增内容(不再要在HTML里有相应的东西)。虽然最终生成的东西并不是真正的DOM里的内容，但这些内容能像普通内容一样显示。

双冒号(::)和单冒号(:)的区别

所有支持CSS3的双冒号(::)语法的浏览器都会支持单冒号(:)语法，但IE8只支持单冒号。建议只使用单冒号，以获得最佳的浏览器支持。双冒号(::)是一种新语法，是用来将伪元素选择器和伪元素区别开。如果不需要IE8支持，就用双冒号(::)吧。

min-width

在页面设计过程中，我们通常使用min-width来设置页面最小宽度，但是IE浏览器却办不到，但是CSS中使用expression完美设置页面最小宽度，并且兼容IE浏览器。CSS中使用expression只有ie才能识别，其把CSS属性和Javascript表达式关联起来，效果相当于min-width，即实现了兼容ie。

如下为解决IE兼容min-width的方法：

                   
body { 
    min-width: 1138px; 
    width: expression((documentElement.clientWidth > 1138) ? "1138px" : "auto" ); 
}

上面例子中min-width是只页面的最小宽度，而ie不支持这种方法，只能用只有ie才识别的expression来解决。

复选框/单选按钮与文本位置水平问题

在三个登陆页面中，登陆框中在ie9及以下浏览器中出现复选框与文本不在同一水平线上的现象。因为在bootstrap.css中有“margin-top: 4px\9;”属性及属性值，“\9”的意思是只有ie9及ie9以下的浏览器才能识别的符号。所以会造成复选框相对“记住我”文字偏下。

怎样解决？

既然是bootstrap.css的原因而且只针对ie9及ie9以下的浏览器，我们可以重写bootstrap.css中有关文本框margin-top的属性值。

我们在组件库中采取同样的解决方式，如下为重写源代码：

                   
.checkbox input[type='checkbox'], .checkbox-inline input[type='checkbox'], 
.radio input[type='radio'], .radio-inline input[type='radio'] {
	margin-top: 0px\9;
}


IE8 prettify.js代码高亮排版问题

使用ie8浏览器查看组件代码高亮时，你可能会发现代码行与行之间不分段的现象，这就是我们接下来要说的ie8下代码高亮问题。

我们在用prettify.js的时候，会使用以下代码：

                   
$(document).ready(function() {                   
    $('pre').each(function() {
        var element = $(this);
        var html = element.html().replace(//g,'>');
        element.html(html);
    });
    $("pre").addClass("prettyprint");
    prettyPrint();
}

在这段代码中忽略了ie8不支持.html()事件，所以会造成一开始说的代码行与行之间不分段的现象。所以在使用.html()之前先对浏览器进行判断：

                   
$(document).ready(function() {
    if(!$.support.leadingWhitespace) {
        $('pre').each(function() { 
            var element = $(this);
            element.text(element.html());
        });
        $("pre").addClass("prettyprint");
        prettyPrint();
    }else {
        $('pre').each(function() {
            var element = $(this);
            var html = element.html().replace(//g,'>');
            element.html(html);
        });
        $("pre").addClass("prettyprint");
        prettyPrint();
    }
});

if(!$.support.leadingWhitespace) {} 意思是判断是否是ie6~ie8。
