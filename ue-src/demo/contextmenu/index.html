js配置方式:

  • HTML
    			


         右键单击此处

js非模块化加载：

需要引用的js文件及加载顺序：

jquery.js，bootstrap.js，form.js

引用方式：

正常js引用方式 <script type="text/javascript" src="jquery.js路径"></script>

  • JS（js非模块化加载）
$('#context2').contextmenu({
    target: '#context-menu',
    onItem: function (context, e) {
        alert($(e.target).text());
    }
});

js模块化加载：

在require之前需要引用的js文件：

require.js，main.js

引用方式：

<script src="require.js路径"></script>

<script src="main.js路径"></script>

之后在require中加载组件所需要的js文件，jqueryform/contextmenu

将js文件放在数组里面，js的引用在非模块化中是无序的，注意文件不要加.js后缀，因为RequireJS会自动加载。

  • JS（js模块化加载）
                
require(['jquery','form/contextmenu'], function($){
    $('#context2').contextmenu({
        target: '#context-menu',
        onItem: function (context, e) {
            alert($(e.target).text());
        }
    });
});

无论DOM配置方式还是JS配置方式都需要引用的css文件：

bootstrap.css，ui.css

说明

不论是在DOM配置方式中还是在JS配置方式中，元素div需要自定义上下文菜单。

DOM配置方式设置data-toggle为context，即作用是设置显示方式为上下文。 如果想在DOM配置方式的盒子中单击右键出现上下文菜单，需要设置data-target属性，作用就是设置触发目标，因此在设置上下文菜单盒子时，千万不要忘了设置与 data-target相对应的id选择器。

DOM的配置方式需要设置DOM结构和上下文所需要的js配置。

.divider 为上下文菜单添加一条分割线。


名称 说明
target 右键弹出的目标菜单id
onItem 菜单单击事件
