Konckout是什么

Knockout是个模板绑定器,简称ko。通过应用MVVM模式使JavaScript前端UI简单化,你可以在任何时候动态更新UI的选择部分。

Knockout有如下4大重要特性:

声明式绑定 (Declarative Bindings)：使用简明易读的语法很容易地将模型(model)数据关联到DOM元素上。



UI界面自动刷新 (Automatic UI Refresh)：当模型状态(model state)改变时，UI界面将自动更新。



依赖跟踪 (Dependency Tracking)：为转变和联合数据，在模型数据之间隐式建立关系。

模板 (Templating)：为模型数据快速编写复杂的可嵌套的UI。


实现原理

采用MVVM模式实现，什么是MVVM模式？

MVVM是model-view-model view的简写。

model是模型，数据访问层。

View是视图，UI界面。

View model 是视图模型,它是view的抽象，负责view与model之间信息交换，将view的实际操作的调用传送到model。


Ko,怎么使用?

简单来说：首先导入knockoout.js文件，声明你的数据作为一个JavaScript 模型对象（model object），然后将DOM 元素或者模板（templates）绑定到它上面.

创建一个view model，只需要声明任意的JavaScript object。例如：

	                     
var myViewModel = {
    personName: 'Bob',
    personAge: 123
};

激活Knockout，让它起作用需要添加如下的 代码块：

                           
ko.applyBindings(myViewModel);

KO一个重要的功能是当view model改变的时候能自动更新界面。 所以需要将model属性声明成observable的,能够通知订阅者它的改变以及自动探测到相关的依赖。

例如：将上述例子的view model改成如下代码：



var myViewModel = {
    personName: ko.observable('Bob'),
    personAge: ko.observable(123)
};

不需要修改view所有的data-bind语法依然工作，不同的是他能监控到变化，当值改变时，view会自动更新。
